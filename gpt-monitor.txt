Totally doable. Here’s the fastest way to verify if you have cluster CPU/RAM capacity metrics (so we can compute “namespace % of total cluster”):
Check in Grafana → Explore (Prometheus data source)
Paste these one by one. If they return a number (not “no data”), you’re good.
Cluster capacity (kube-state-metrics style):
sum(kube_node_status_capacity{resource="cpu",   unit=~"core|cores"})
sum(kube_node_status_allocatable{resource="cpu",unit=~"core|cores"})

sum(kube_node_status_capacity{resource="memory",   unit="byte"})
sum(kube_node_status_allocatable{resource="memory",unit="byte"})
If you’re on an older KSM, try the legacy forms (sometimes exposed by distros):
sum(kube_node_status_capacity_cpu_cores)
sum(kube_node_status_allocatable_cpu_cores)
sum(kube_node_status_capacity_memory_bytes)
sum(kube_node_status_allocatable_memory_bytes)
These metrics come from kube-state-metrics and represent per-node capacity/allocatable; summing gives cluster totals.
